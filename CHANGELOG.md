### [3.35.0] - 2026-07-11
- **ADD**: New skill `hover-effects` (78th skill) — pointer-driven micro-interaction recipes for buttons/cards/links, organized by technique: `cursor-effects.md` (CSS custom-prop pointer tracking: ripple, spotlight/lantern, magnetic, gradient-border follow, 3D tilt), `canvas-effects.md` (scratch-off foil via `destination-out`, particle burst/trail), `svg-effects.md` (grass blades, thermal heat via `feTurbulence`, DNA helix), `motion-effects.md` (Motion springs: shine sweep, press physics, glitch, text stagger/scramble). Added to `frontend-specialist`'s frontmatter skills. Scaffolded via `scaffold_new_skill.py` after `verify_similar_skills.py` confirmed no duplicate (closest real neighbors are `frontend-design`'s animation-guide/visual-effects, which cover when/why to animate, not pointer-effect construction).
- **IMPROVE**: `brainstorming` skill's Socratic Gate gained an **Intent Extraction Matrix** (adapted from `nidhinjs/prompt-master`, logged in `pattern-mining-log.md` 2026-07-11): before generating questions, silently extract 9 dimensions (goal, users, scope, success criteria, constraints, existing context, content/input, output shape, references) and ask **only about missing critical dimensions** (max 3 per round), instead of firing the same fixed Purpose/Users/Scope trio by reflex — that trio is now the fallback when the matrix yields nothing more specific. Already-answered, codebase-inferable, or `question-preferences.md`-suppressed dimensions are filled with a declared assumption, not asked. DEVBUREAU.md's P0 minimum (3 questions for new builds) unchanged — spare slots fill with edge-case/trade-off questions. Outbound prompt-generation from the same repo (per-tool templates for Midjourney/Cursor/etc.) deliberately skipped per Regra dos Três — no demonstrated demand yet.
- **NOTE**: Origin — user shared an Instagram-style "Hover Button Effects Pt.5" image attributing it to the `suraniharsh/CodeCandy` GitHub repo; investigation (clone + grep) showed that repo is a snippet-manager app containing none of those effects, and the image's CSS is illustrative pseudo-code. That misattribution + pseudo-CSS trap is the documented RED pressure scenario per `writing-skills` (see `.agent/memory/gotchas.md`, 2026-07-11). `GUIA_DO_USUARIO.md` intentionally not touched, matching the v3.33.0 `config-gc` precedent for agent-internal skills.
