- **FIX**: Python `__pycache__`/`.pyc` files generated locally by running `doctor.py`/`pytest` were leaking into the published npm tarball (confirmed via `npm pack --dry-run` on the live 3.20.0 package — a 33.6kB `.pyc` was sitting inside the published `.agent/tests/`). Kept `.npmignore` as a documented (non-functional, by npm's own behavior) belt-and-suspenders note pointing at the real fix.
- **NOTE**: 3.20.0 was left published as-is rather than unpublished — its code is correct, only the registry's readme selection and a packaging leak were wrong, and `npm unpublish` would have permanently blocked reusing the version number `3.20.0` for no real benefit.
